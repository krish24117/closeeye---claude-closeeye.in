Design iteration · nothing shipped · the understanding loop

Never show someone a question
they aren’t allowed to answer.

The loop is right: you type, Connect understands, it asks for what it’s missing. The screens are what break it — today it lists four questions you cannot touch, and hides the key behind a button about something else.

What happens now

s1 reveals the understanding and lists the open questions — as dead text. “Which city Hospital is in”, “His health, and what he manages day to day”. You cannot answer any of them. The only way forward is “That’s exactly it” — a button that confirms the reading, not the questions. Pressing it lands you on s2: the same list, same place, same wording — except now the circles are tappable. Two near-identical screens, one silent mode switch. The user is asked to guess that a confirmation is also a key.

“Not quite” must land somewhere. Today it’s a flag with no repair. Naming the three things that can be wrong turns a complaint into a correction — and each one is a slot the engine already has, so the fix is real, not sympathy.

Why this can’t confuseFour moves, one deletion

01

Delete the dead questions from s1

The single highest-value change here is a removal. A question you can’t answer is worse than no question: it teaches people the screen is decorative.

02

Merge the mode switch away

s1 and s2 look identical and behave differently. Ask one thing per screen and the switch stops existing — nobody has to notice a transition that no longer happens.

03

Open the first box for them

Don’t make someone tap a circle to discover it’s a button. The first question is already open, with the cursor in it. The rest queue behind it.

04

“That’s enough for now” is always there

Plus a count and a bar. Never trapped, always knows how much is left, and skipping costs nothing — the ○ simply stays a ○, honestly.

The circles aren’t a checklist. They’re the product: a menu that gets shorter the better it knows you — which is the opposite promise to a chatbot’s empty box.

Honest notesBefore any of this is built

“A Presence Manager takes it from here” is a promise, not a description. It is the one line in this design that isn’t already true on the screen — it needs to be staffed on day one or it becomes exactly the false-capability class we spent a day deleting. Your call, not mine.

The repair screen (D) invents nothing. “It’s not my mother” / “she doesn’t live alone” / “that’s not what I need” map to the subject, situation and need slots the engine already fills — so a correction can actually re-run understanding rather than just log a complaint.

What I did not touch: the paper, the serif, ✓/○, “from your words” vs “my reading”. That vocabulary is the trust, and it’s already right. Everything above is a rearrangement of screens, not a redesign of the idea.
