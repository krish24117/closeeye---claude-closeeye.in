The model understands language.
Close Eye owns trust, policy, and execution.

The canonical request flow for every domain — Health, Finance, Legal, Home, and whatever comes next. An LLM doing only language, bracketed by deterministic gates on both sides, with two exits that never reach it and one lane it can route back into.

The invariant

Safety and execution policies never depend solely on the LLM. The model understands nuance, context and indirect requests; Close Eye decides what it can do, whether it may, carries it out, and remembers.

What changed in this revisionFour completions

◆ 01

A Response Composer

The flow understood the family but never spoke back. The LLM's second role — communication — phrases the reply, constrained to what the deterministic layers decided. Propose/dispose, on the way out too.

◆ 02

Outside Scope short-circuits

A denial terminates at the router — it never flows through pricing and workflows. Same shape as the crisis exit: two paths leave before the Decision Engine.

◆ 03

Safety is a gate AND a lane

The front gate catches explicit danger. But the LLM surfaces the indirect kind ("he's taken a lot of his pills") — so an intent of emergency re-enters the Safety lane rather than proceeding.

◆ 04

The Context Builder is a trust boundary

The gate checks the user's input. Retrieved documents can carry injection the gate never saw, so what gets built into the prompt is trust-checked at assembly.

Why it's a principle, not a fixIt holds as Close Eye grows

Putting the boundary after structured intent — not on raw text — is what survives expansion. A new domain plugs into the same spine: the LLM understands its language, the Capability Router gains a few entries, the Decision Engine gains its coverage rules, the Orchestrator gains an actor. The gateway, retrieval, the human-owned policy layer, and memory are unchanged. One skeleton, many domains.

GUARDRAIL 01

Classify, don't compose

A deterministic router is only as trustworthy as its input. The LLM picks from a closed set with an unsure → human value — never free-form structure. That bucket turns the model's uncertainty into a safe default, not a confident wrong guess.

GUARDRAIL 02

The model never holds a policy

Two LLM stages only — understanding (in) and composing (out). Between them, everything that decides what, whether, and to whom is Close Eye's deterministic code.

Honest statusWhat of this is built, today

Eight of ten stages are Close Eye's deterministic own; two are the LLM. Built and live: the Gateway (identity + safety), the Decision Engine, coverage. The keystone gap is Retrieve — it unblocks everything from the Context Builder onward. Near-term order: Retrieve → LLM understanding + structured intent → Capability Router → Response Composer. Safety, decision and coverage already stand; today's Connect runs a deterministic stand-in for the LLM understanding step.
